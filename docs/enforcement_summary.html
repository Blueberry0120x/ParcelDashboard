What This System Does

ControllerAuto-enforces coding standards across 12+ repos

19 hooks fire at 5 lifecycle points — no manual steps. They block bad commits, warn on policy drift, auto-clean stale files, prevent secret leaks, and enforce cross-platform compatibility.

Five Lifecycle Phases

1 · Session Start3 hooks always · 1 on compaction

Checks rule files exist (blocking), cleans stale artifacts, scans pings, verifies hook/skill inventory and XPLAT-001.

2 · Pre-Bash7 hooks · all potential blockers

Runs before every shell command. Blocks find.exe, validates branch names, enforces PS StrictMode and COM lifecycle on commit, enforces XPLAT-001, prevents duplicate remote sessions.

3 · Pre-Edit/Write2 hooks · warn only

Auto-backs up config files. Reminds to trace hook chains before editing hook files.

4 · Post-Tool5 hooks · warn / notify / auto

Scans for leaked secrets, reminds to write upnotes after commits, announces unread pings, sends GitHub notifications for major ops, auto-touches .ping on upnote writes.

5 · Session Stop1 blocker · 1 auto-commit

Blocks session end if any repo has unread pings. Auto-commits any pending changes so no work is silently lost.

Hook Changes (3)

Fixpre_commit_guard.pyDone

Ping blocking at commit was wrong timing. The rule is "don't END a session with unread pings" — not "don't commit with them." ping_check at Stop is the gate. Blocking pre-commit added friction with no safety benefit.

− check unread ping → BLOCK
− _parse_ts() / datetime imports
+ auto-clean stale .bak/.tmp only
Dedupsettings.jsonDone

single_instance.py was wired at BOTH SessionStart and PreToolUse. Same file, same logic. SessionStart fires before any action is possible — the PreToolUse wire (on --start) is the correct and only gate.

− SessionStart: single_instance.py
PreToolUse: single_instance.py ← kept
Deduprules_memory_check.pyDone

Both session_guard and rules_memory_check checked that CLAUDE.md and global_rules.md exist. session_guard owns the blocking existence gate. rules_memory_check now owns only its unique work.

− check CLAUDE.md exists
− check global_rules.md exists
+ XPLAT-001 content check ← unique
+ all hooks .py exist? ← unique
+ all skills .md exist? ← unique
Bug Fixtest_remote_invoke.pyDone

Pre-existing failure. Test mocked subprocess.run but not _is_pid_alive, which uses ctypes directly. A real .pid file in tools/ was alive — ctypes saw it and added a ghost second result.

+ @patch("…_is_pid_alive", return_value=False)
PID-file fallback isolated from live env
411 tests pass (was 1 failure)
