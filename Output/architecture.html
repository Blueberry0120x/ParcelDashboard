Port 3034
Serves the compiled HTML and handles save/switch API calls. Browser talks to this.
MethodPathWhat it does
GET / Serve InteractiveMap.html
GET /checklist Serve PreApp_Checklist.html
GET /api/sites List all sites + which is active
POST /save Write .saved to per-site file, preserve .site, rebuild HTML
POST /api/sites/{id}/activate Update activeSiteId pointer, rebuild HTML
POST /api/sites/{id}/update-site Edit whitelisted .site fields, rebuild HTML
POST /backup-checklist Archive checklist JSON to config/backup/

Engine Breakdown
src/js/
EngineLinesWhat it ownsDepends on

The Save Flow (Most Important Path)
6
ExportEngine.save() -- The ONE Save Function
Critical Path
Every button that changes state calls this. It's the single write path -- no exceptions.
Step A
Sync UI to State
Read all inputs → write to ConfigEngine.state
Step B
_payload()
Build canonical save object from state
Step C
localStorage
Write site_state key (instant, offline)
Step D
POST /save
Send to server (if localhost)
Step E
Server Writes JSON
Preserve .site, overwrite .saved
Step F
Rebuild HTML
Both files recompiled with new state
What's in _payload():
lat, lng, rotation, locked
buildings[] (W, D, offset, count, stacking...)
setbacks (front, rear, sideL, sideR)
vehicles[], activeVehicle
commFront, showBldgDims, hiddenDimKeys
freeDrag, snapEdge, mapOpacity
chainWOffset, chainDOffset
checklist (if present)
